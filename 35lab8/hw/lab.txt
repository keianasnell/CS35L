access -- check real user's permissions for a file
open -- open and possibly create a file or device
fstat -- get file status
close -- close a file descriptor
read
mmap -- map or unmap files or devices into memory
mprotect -- set protection on a region of memory

Access checks the permissions for editing the file. 
Open and fstat get information about the library. 
Mmap creates the map for the library, and read reads the data info about it. 
Mprotect prevents memory from being modified as it's mapped; close closes the library once finished.

I determined that the commands that ldd could not find libraries for were for commands that were shell
 or Perl scripts that were not using any dynamically linked libraries. 
This is why they are not dynamic executables.
I did this using file /usr/bin/<command>:
/usr/bin/xml2-config: POSIX shell script, ASCII text executable
/usr/bin/rxe_cfg: Perl script, ASCII text executable
/usr/bin/pomigrate2: Bourne-Again shell script, ASCII text executable
/usr/bin/neqn: POSIX shell script, ASCII text executable
/usr/bin/cpanp-run-perl: Perl script, ASCII text executable
